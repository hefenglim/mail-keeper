# Feature Specification: 輸出體驗優化 —— 檔名自動補副檔名、處理進度條、Excel 相容 CSV 編碼

**Feature Branch**: `004-output-ux-progress`

**Created**: 2026-06-23

**Status**: Draft

**Input**: User description: "該版本修正，進行以下三點優化改進：(1) 所有功能的檔名輸入若沒有附檔名，必須自動補上（inbox → inbox.csv）。(2) 新增一個獨立的『狀態進度條』模組，用在任何有機會處理大量資料的迴圈，即時更新處理進度，避免用戶誤以為當機；請重新審查目前所有處理動作的迴圈並套用。(3) 輸出的 CSV 在 Notepad 正常但用 Excel 開啟時中文亂碼，請改善編碼不通用的問題。"

本功能為 v0.4.0 之後的優化修正版（v0.5.0），不新增選單功能、不更動 CSV 欄位結構、不更動 OAuth 流程，只打磨既有功能 1～3 與其共用的輸出體驗。

## Clarifications

### Session 2026-06-23

- Q: 匯出 CSV 的編碼策略（Excel 相容 vs AI/LLM 解析）？ → A: 採 **UTF-8 + BOM（utf-8-sig）**；讀取端容忍並剝除 BOM。
- Q: 大量郵件下載時進度如何前進？ → A: **下載期間就分批前進**（標頭讀取改為分批 fetch，網路下載過程即時更新進度，而非下載完成後才更新）。
- Q: 進度顯示的套用門檻？ → A: 任何迴圈待處理項目數**超過 30（> 30）**時即顯示即時進度；30 以內不顯示。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - CSV 在 Excel 開啟不亂碼（編碼通用）(Priority: P1)

使用者用功能 1／2 匯出 CSV 後，常用 Microsoft Excel 開啟檢視與編輯。目前檔案在 Notepad 顯示正常（中、英文都看得到），但用 Excel 開啟時中文完全亂碼，導致匯出的資料在最常用的工具裡無法閱讀。使用者希望匯出的 CSV 不論用文字編輯器或 Excel 開啟，各國語文都正確顯示。

**Why this priority**: 這是「輸出正確性」缺陷 —— 匯出的檔案在最主流的試算表工具中不可讀，等同核心輸出失效，影響最大，最該優先修。

**Independent Test**: 匯出一份含中／英／日／韓／阿拉伯文與 emoji 的工作表 CSV，分別用 Excel 與文字編輯器開啟，確認所有字元正確顯示、無亂碼；再把該檔交給功能 3 重新匯入，確認仍可正確解析。

**Acceptance Scenarios**:

1. **Given** 一個含多國語文的資料夾，**When** 使用功能 1 匯出工作表 CSV 並用 Microsoft Excel 開啟，**Then** 所有中文與其他語文字元正確顯示、無亂碼。
2. **Given** 同一份 CSV，**When** 用純文字編輯器（如 Notepad）開啟，**Then** 內容仍正確顯示、無多餘符號干擾閱讀。
3. **Given** 一份由 Excel 另存後可能帶有編碼標記（BOM）的工作表，**When** 功能 3 重新匯入該檔，**Then** 標頭與每一列都被正確解析、不因編碼標記而出現殘留字元或欄位錯位。

---

### User Story 2 - 檔名未填副檔名時自動補上 (Priority: P2)

使用者在任何功能的檔名輸入（互動式提示或命令列路徑參數）只輸入名稱、沒有打副檔名時（例如輸入 `inbox`），系統應自動補上 `.csv`，使輸出檔成為 `inbox.csv`，避免產生一個沒有副檔名、雙擊無法以試算表開啟的檔案。

**Why this priority**: 目前輸入 `inbox` 會真的存成名為 `inbox`（無副檔名）的檔，違反使用者預期且不易開啟；屬輸出正確性／易用性缺陷，但比 P1 影響範圍小。

**Independent Test**: 在功能 1／2 的輸出路徑提示輸入不含副檔名的名稱，確認實際寫出的檔名自動帶 `.csv`，且確認訊息顯示的是補完後的檔名；輸入已含副檔名者則保持不變。

**Acceptance Scenarios**:

1. **Given** 功能 1 的輸出路徑提示，**When** 使用者輸入 `inbox`（無副檔名），**Then** 系統寫出 `inbox.csv`，且完成訊息顯示實際檔名 `inbox.csv`。
2. **Given** 任一檔名輸入，**When** 使用者輸入已含副檔名的名稱（例如 `inbox.csv`），**Then** 系統保持原檔名、不重複附加（不會變成 `inbox.csv.csv`）。
3. **Given** 功能 3 的輸入路徑，**When** 使用者輸入不含副檔名的名稱，**Then** 系統以補上 `.csv` 後的檔名尋找並讀取。

---

### User Story 3 - 大量處理時顯示即時進度，避免誤判當機 (Priority: P3)

使用者對「擁有大量郵件的資料夾」執行匯出或分類搬移時（例如 426 封郵件），過程可能耗時數秒到數十秒。目前畫面在處理期間沒有任何回饋，使用者會懷疑程式是否當機。系統應提供一個可重複使用的「進度條／狀態」能力，在任何可能處理大量項目的迴圈中即時更新進度（目前數／總數或百分比），讓使用者明確知道「正在處理中」。

**Why this priority**: 屬使用體驗（感知回應性）改善，不影響資料正確性；在 P1／P2 修好後再加，價值與獨立性都成立，故列為 P3。

**Independent Test**: 對一個大資料夾（如 400+ 封）執行匯出與分類搬移，確認過程中進度資訊持續更新（不是只在結束時才出現），且在非互動（導向檔案／管線）情境下不破壞主要輸出、不卡住。

**Acceptance Scenarios**:

1. **Given** 一個含大量郵件的資料夾，**When** 使用功能 1 匯出，**Then** 過程中持續看到處理進度即時更新（目前／總數或百分比），結束時進度乾淨收尾。
2. **Given** 一份含多列待搬移的工作表，**When** 執行功能 3 的實際搬移，**Then** 每搬移一批／一封即更新進度，使用者可見處理進展。
3. **Given** 以非互動方式執行（輸出被導向檔案或管線、非終端機），**When** 任一含進度的操作執行，**Then** 進度顯示自動降級（不輸出動畫控制字元），且不污染 CSV／資料輸出、不阻塞。

---

### Edge Cases

- **檔名邊界**：輸入以點結尾（`report.`）、隱藏檔式名稱（`.gitignore`）、含多個點（`a.b.c`）、或已含非 `.csv` 副檔名（`data.txt`）時的判定 —— 已有副檔名者保持不變，不被改成 `.csv`；確實無副檔名者才補 `.csv`。
- **空輸入**：使用者於提示直接按 Enter，沿用既有預設檔名（如 `worksheet.csv`），不受本功能影響。
- **編碼標記（BOM）往返**：由 Excel 另存的檔可能新增或保留 BOM；功能 3 讀取時必須能容忍有無 BOM，皆不在第一欄／標頭留下殘留字元。
- **進度總數未知**：若某操作無法事先得知總數，進度應以「已處理數／不定量」方式呈現，不得崩潰。
- **進度與錯誤並存**：迴圈中途某項失敗時，進度條須乾淨收尾、不殘留半截，且錯誤仍依既有錯誤處理浮現（不崩潰、不卡死）。
- **小量資料**：對極少量項目的操作，進度顯示不應造成干擾（可不顯示或極簡顯示）。

## Requirements *(mandatory)*

### Functional Requirements

**編碼（US1）**

- **FR-001**: 系統寫出的所有 CSV 檔，MUST 在常見試算表軟體（如 Microsoft Excel）與純文字編輯器中皆正確顯示，涵蓋至少 5 種語文（含中日韓、阿拉伯文）與 emoji，無亂碼。
- **FR-002**: 系統讀回先前匯出的工作表（功能 3）時，MUST 能正確解析「有或沒有編碼標記（BOM）」的檔案，且不得在第一欄或標頭留下殘留字元或造成欄位錯位。
- **FR-003**: 編碼調整 MUST 維持既有固定欄位順序與內容、不得遺失或竄改資料。

**檔名副檔名（US2）**

- **FR-004**: 對使用者提供的每一個 CSV 檔名（互動式提示與命令列路徑參數，含讀取與寫入），若該名稱不含副檔名，系統 MUST 在使用前自動補上 `.csv`。
- **FR-005**: 若使用者已提供副檔名（`.csv` 或其他），系統 MUST 保持原樣、不得重複附加（不得產生 `name.csv.csv`）。
- **FR-006**: 系統 MUST 在確認／完成訊息中顯示補完後的實際檔名，使使用者看到真正寫出或讀取的檔案。

**進度（US3）**

- **FR-007**: 系統 MUST 提供一個可重複使用、獨立的「進度／狀態回報」能力，供任何可能處理大量項目的迴圈使用。
- **FR-008**: 會「逐封讀取資料夾郵件標頭」與「逐封搬移／分類郵件」的操作 MUST 在執行期間顯示即時更新的進度（目前數／總數或百分比）。
- **FR-009**: 進度 MUST 於迴圈進行中逐步更新（而非僅於結束時出現），使使用者能看見正在處理。
- **FR-010**: 當輸出非互動終端機（被導向檔案／管線／非 TTY）時，進度顯示 MUST 優雅降級，不得污染主要（CSV／資料）輸出，且不得阻塞。
- **FR-011**: 進度資訊 MUST 走不污染資料輸出的通道（狀態走主控台／stderr，資料走檔案），與既有「防崩潰安全輸出」相容。
- **FR-012**: 進度於完成或發生錯誤時 MUST 乾淨收尾（不殘留半截），且錯誤仍須循既有錯誤邊界浮現，全程不崩潰、不卡死。
- **FR-015**: 進度 MUST 僅在迴圈待處理項目數「超過 30」（> 30）時顯示；30 以內 MUST NOT 顯示進度，以免小量操作受干擾。
- **FR-016**: 「逐封讀取資料夾郵件標頭」MUST 以分批方式取得，使進度於網路下載過程即時前進（MUST NOT 僅於下載完成後才更新）。

**通用約束**

- **FR-013**: 本功能 MUST NOT 在 `MailBackend` 介面層之上引入後端細節（維持後端隔離原則）；進度回報須以後端中立方式進行，不得使既有契約耦合特定協定。
- **FR-014**: 本功能 MUST NOT 變更 CSV 欄位結構、選單功能集合或認證流程；僅優化既有功能 1～3 的輸出與處理回饋。

### Key Entities *(include if feature involves data)*

- **進度回報器（Progress reporter）**：代表一個進行中的長時操作，具「已處理數」與「（可選）總數」，可被任何迴圈以「開始／更新一步／結束」的方式驅動；負責決定是否顯示、以何種頻率刷新、以及在非互動環境如何降級。
- **CSV 檔（輸出／輸入）**：功能 1／2 寫出、功能 3 讀回的工作表與資料夾清單；本功能調整其「寫入編碼」使其在試算表與文字編輯器皆通用，並使讀取端能容忍編碼標記。
- **檔名（Filename）**：使用者提供的輸出／輸入路徑；本功能在缺副檔名時補上 `.csv`，並回報補完後的實際值。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 以工具匯出的 CSV，用 Microsoft Excel 開啟時，至少 5 種語文（含中日韓、阿拉伯文）與 emoji 之字元 100% 正確顯示、無亂碼。
- **SC-002**: 使用者於檔名輸入未填副檔名時，寫出的檔案 100% 以 `.csv` 結尾；已填副檔名者 100% 維持原樣、無重複副檔名。
- **SC-003**: 對超過 30 封（如 400+）郵件的資料夾執行匯出或搬移時，使用者在處理期間（含網路下載階段、分批前進）能看到進度持續更新，且不出現超過數秒、令人懷疑當機的無回饋空窗；30 封以內則不顯示進度。
- **SC-004**: 「匯出 → 以 Excel 另存 → 重新匯入功能 3」的往返，不論檔案是否含編碼標記（BOM），皆以 0 解析錯誤完成。
- **SC-005**: 在新行為下所有操作維持零崩潰、零卡死：非互動執行可順利完成、不阻塞；迴圈中途失敗時進度乾淨收尾且錯誤正常浮現。

## Assumptions

- 「沒有副檔名」定義為：解析後的檔名不具非空副檔名後綴；補上的預設副檔名為 `.csv`（三個功能皆以 CSV 為輸出）。使用者明確提供的非 `.csv` 副檔名予以尊重、不覆寫。
- **（2026-06-23 釐清確認）** Excel 相容性以「UTF-8 並加上位元組順序標記（BOM，即 UTF-8-sig）」達成：純文字編輯器與 Python／多數 CSV／LLM 解析器皆能容忍 BOM，且讀取端會剝除 BOM。取捨：相較無 BOM 的純 UTF-8，會在檔首多一個 BOM，但換取主流 Excel 直接正確判讀中文，以使用者實際使用情境為優先。
- 進度顯示於互動終端機透過主控台／stderr 呈現；非互動（管線／重導向／非 TTY）情境降級為極簡或不顯示，且不輸出終端控制字元、不阻塞。**（2026-06-23 釐清）進度門檻：待處理項目數 > 30 才顯示，30 以內不顯示；標頭讀取改以分批取得，使網路下載期間進度即時前進。**
- 進度能力為後端中立（以回呼／中介機制回報），不在 `imap_client` 之外引入 `imaplib`，維持憲法 Principle I 的後端隔離。
- 範圍：本功能建立於 feature 003（`csv_io`／`classifier`／`cli`／`menu`）與 001（防崩潰安全輸出）之上，屬其優化；不新增選單功能、不變更欄位結構與認證。

## Dependencies

- 依賴既有功能 003（CSV 匯出／分類）與 001（跨平台防崩潰輸出層）。
- 進度與檔名補完之行為，套用於既有 `cli` 的互動提示與子指令路徑，以及 `csv_io` 的讀寫與處理迴圈。
